## 对话要点总结

### 项目背景深度挖掘

#### 核心问题定位
- **替代 Jenkins**: 用户明确表示 Jenkins 过重，无法很好与容器配合使用
- **轻量化需求**: 需要一个轻量化的集构建发布一体的引擎工具
- **UI 层预备**: 为后续实现 UI 层调用此工具做准备

#### 技术选型理由
- **Rust 选择**: 作为引擎，Rust 更稳定，性能更好，"值得"投入
- **个人项目**: 工作之余开发，大约一个月时间
- **满足需求**: 已经可以满足个人绝大部分使用场景

### 具体使用场景分析

#### 构建产物特点
- **多样化产物**: 主要是运行时脚本或者二进制文件
- **分发方式**: 上传至服务或者直接发布各类执行文件到发布平台
- **个人环境**: 基于自己的环境做产物构建，不追求通用性

#### 技术架构决策
- **双引擎支持**: Docker/Podman 支持主要是为了方便在两个工具之间切换使用
- **双模式设计**: 
  - 交互模式：提供直接使用
  - 命令模式：为了方便日后开发 Dashboard 对接

#### 当前状态评估
- **基础功能**: 基于配置各种场景发布目前都能满足
- **事件系统**: 暂时没有实现，直接通过脚本控制，后续会支持
- **优势定位**: 优势不在于他人，而是基于自己的环境做产物构建

### 关键技术洞察

#### 个人定制化 CI/CD
- 不追求通用性，专注个人工作流优化
- 轻量化替代重型 Jenkins 的明确目标
- 为后续 Dashboard UI 预留架构空间

#### 实用主义导向
- 一个月快速开发，满足实际需求
- 优先实用性而非完美性
- 基于真实使用场景迭代优化

## 项目定位确认

### 核心价值主张
- **个人定制化**: 基于个人环境和习惯的定制化 CI/CD 引擎
- **轻量高效**: 替代重型 Jenkins，提供轻量化解决方案  
- **架构前瞻**: 为后续 Dashboard UI 开发奠定引擎基础
- **双模式支持**: 兼顾直接使用和程序集成需求

### 技术特色
- **Rust 实现**: 保证性能和稳定性的技术选型
- **双引擎支持**: 灵活适配 Docker/Podman 使用习惯
- **配置驱动**: 支持各种发布场景的灵活配置
- **产物多样性**: 适应不同类型项目的构建需求
