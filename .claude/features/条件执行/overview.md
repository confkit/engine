## 功能目标
为 confkit 配置执行系统添加动态条件执行能力，支持 steps 根据环境变量进行条件判断，以及 environment_from_args 的累积式交互

## 用户价值
- 提供灵活的 CI/CD 流程控制，根据不同环境和条件执行不同步骤
- 支持复杂的多分支场景，避免创建多个重复配置文件
- 实现交互式配置的渐进式决策，前面的选择影响后续选项
- 简化配置管理，一个配置文件适配多种执行场景

## 核心功能
- Step 级别的条件执行支持
- 简单条件表达式解析引擎
- Interactive 环境变量累积机制
- 环境变量动态条件判断
- 条件表达式错误处理和中断机制

## 功能边界

### 包含的功能
- 在 ConfKitStepConfig 中添加可选的 condition 字段
- 支持基本的比较运算符（==, !=, >, <, >=, <=）
- 支持逻辑运算符（&&, ||, !）
- 支持环境变量引用语法 ${VAR_NAME}
- Interactive 模式下环境变量的累积传递

### 不包含的功能  
- 复杂的函数调用和自定义表达式
- 条件执行的性能优化和缓存
- 条件执行的可视化调试工具
- 与外部系统的条件集成

## 使用场景
- 根据环境变量 ${NODE_ENV} 选择不同的构建步骤
- 基于用户交互选择的部署目标执行不同的部署脚本
- 根据 Git 分支名决定是否执行测试步骤
- Interactive 模式下根据前面选择的服务类型显示相关配置选项
