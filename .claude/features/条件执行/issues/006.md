# 条件执行日志和可见性

## 议题目标
实现条件执行的日志记录和用户可见性

## 实现要点
- 记录条件表达式和评估结果
- 跳过步骤的原因说明
- 条件解析错误的详细日志
- 统计信息显示 (执行/跳过/失败步骤数)

## 技术细节
基于现有的事件驱动日志系统（EventHub + TaskLogger + LogSubscriber），在 StepExecutor 中增加条件相关的日志记录。通过 TaskLogger 发布条件执行事件，LogSubscriber 负责格式化输出和文件记录。

现有日志架构集成：
1. 使用 TaskLogger::info()、error() 方法记录条件相关的任务日志
2. 使用 tracing::debug()、tracing::trace() 记录调试信息和详细过程
3. 通过 EventHub 发布条件执行相关事件到任务日志文件
4. LogSubscriber 自动处理任务日志格式化和文件写入
5. 在 StepExecutor::execute_step() 中增加条件评估日志
6. 扩展 log_step_details() 方法显示条件表达式信息

日志增强内容：
1. 步骤开始时显示条件表达式（TaskLogger::info 记录到任务日志）
2. 条件评估结果和耗时（tracing::debug 输出到控制台）
3. 跳过步骤的明确原因说明（TaskLogger::info 记录到任务日志）
4. 条件解析错误的详细错误信息（TaskLogger::error 记录到任务日志）
5. 任务结束统计：总步骤数、执行数、跳过数、失败数
6. 调试级别显示条件评估详细过程（tracing::trace 输出）
