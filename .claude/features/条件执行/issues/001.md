# 核心配置数据模型扩展

## 议题目标
在现有配置结构中添加条件字段支持，为条件执行功能提供数据模型基础

## 实现要点
- 在 ConfKitStepConfig 中添加 condition: Option<String> 字段
- 在 ConfKitEnvironmentInteractiveConfig 中添加 condition: Option<String> 字段
- 确保序列化/反序列化正常工作
- 添加必要的默认值处理函数
- 保持向后兼容性

## 技术细节
需要在 types/config.rs 文件中进行以下修改：

1. 在 ConfKitStepConfig 结构体中添加字段：
   ```rust
   #[serde(default)]
   pub condition: Option<String>,
   ```

2. 在 ConfKitEnvironmentInteractiveConfig 结构体中添加字段：
   ```rust
   #[serde(default)]
   pub condition: Option<String>,
   ```

3. 确保使用 #[serde(default)] 属性实现向后兼容性
4. 为新字段编写相应的单元测试
5. 验证 YAML/JSON 配置文件的序列化和反序列化功能正常
