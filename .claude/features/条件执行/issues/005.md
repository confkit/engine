# StepExecutor条件集成

## 议题目标
将条件评估集成到现有StepExecutor执行流程中

## 实现要点
- 在步骤执行前增加条件检查
- 条件为 false 时跳过步骤执行
- 条件解析错误时记录错误并继续执行
- 保持现有执行逻辑不变

## 技术细节
修改 `src/core/executor/step_executor.rs` 的 execute_step 方法，在命令执行前增加条件评估。如果条件为 false，设置 StepStatus::Skipped 并直接返回。条件解析失败时记录警告日志但不中断执行。

集成要点：
1. 在 StepExecutor::execute_step 开始处增加条件检查
2. 条件为 None 时按现有逻辑执行
3. 条件为 false 时设置跳过状态并记录日志
4. 条件解析失败时记录错误但继续执行
5. 保持超时、错误处理等现有逻辑完整性
6. 确保事件系统正确触发跳过事件
