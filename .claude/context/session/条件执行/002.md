# 议题 002: 条件表达式词法解析器 - 会话上下文

## 议题概述
实现基于 nom 的条件表达式词法解析器，支持变量、字面量、运算符解析。

## 分析结果
- **技术方案**: 使用 nom 解析器组合子，分阶段实施
- **风险评估**: 中等复杂度，主要风险在运算符优先级和错误处理
- **预估工作量**: 7-8 个工作日
- **实施策略**: 4个阶段增量开发

## 技术决策记录
- **解析器选择**: nom (零拷贝、组合子模式、性能优秀)
- **模块结构**: 在 core/condition/ 下新增 parser.rs
- **错误处理**: 使用 thiserror 包装自定义错误类型
- **优先级处理**: 逻辑或 < 逻辑与 < 比较运算符 < 主表达式

## 已完成功能

- ✅ 字面量支持: 字符串 `'value'/"value"`、数字 `42/-3.14`、布尔值 `true/false`
- ✅ 比较运算符: `==`, `!=`, `>`, `<`, `>=`, `<=`
- ✅ 逻辑运算符: `&&`, `||`
- ✅ 括号分组: `(expression)`
