# Rust Deep Audit

## 重点关注

- panic、越界、terminal restore 失败
- 空转循环、无意义 redraw、事件循环失控
- 状态堆积、抽象失真、主链路阻塞
- 热路径分配、重复遍历、错误异步模型

## 核心检查项

### 崩溃与异常风险

- 是否存在无理由 `unwrap()`、`expect()`、索引越界或 panic 风险点
- 退出路径是否统一, terminal restore、raw mode 恢复是否可靠
- 清理逻辑是否依赖理想路径, 异常中断后是否可能遗留脏状态
- 错误是否可定位, 是否把严重问题吞掉或延迟暴露

### 循环、异步与主链路

- 是否存在空转循环、忙轮询或无事件时仍持续工作
- `loop` / `while` 是否缺少阻塞点、退出条件和清理路径
- 是否用轮询代替事件驱动、channel、stream、`tokio::select!`
- 长耗时任务是否阻塞主线程、主循环、渲染链路或输入处理

### TUI / CLI

- 是否无条件 redraw, redraw 是否仅由状态变化或 tick 触发
- tick 频率是否过高, 是否在终端 IO 链路做无意义刷新
- 输入、状态更新、渲染、IO 是否分层清晰
- 错误输出、日志输出是否可能破坏 terminal 状态或交互体验

### 状态、结构与抽象

- 是否把所有状态堆进巨型 `App` 或单一模块
- UI state、domain state、cache 是否分层, 是否存在重复数据源
- 是否用过深调用链、动态分发或空洞抽象掩盖边界问题
- 模块职责、依赖方向、扩展点是否清晰, 是否违背长期可维护性

### 性能与资源风险

- 热路径是否频繁创建 `String`、`Vec`、错误对象或中间集合
- 是否缺少缓冲区复用、layout cache、字符串缓存和计算结果缓存
- `clone()`、`to_string()`、`format!()`、`collect()` 是否出现在高频路径
- 是否存在重复遍历、终端 IO 过重或主链路日志污染

## 工具建议

- `cargo flamegraph`
- `perf`
- `tokio-console`
