# TypeScript Deep Audit

## 重点关注

- 白屏、交互卡死、运行时异常和失控 Promise
- 状态重复源、重复渲染、异步竞态和主线程长链路
- 页面过重、模块职责失衡、边界混乱
- 高频事件、重复请求、重复计算和对象抖动

## 核心检查项

### 崩溃与异常风险

- 是否存在空值访问、越界访问、未处理异常或 Promise 拒绝泄漏
- 错误边界、兜底状态、异常分支是否清晰, 是否会直接白屏或卡死
- 输入、网络、缓存、反序列化失败后是否能稳定退化
- 是否把关键失败吞掉, 导致问题只表现为无响应或脏状态

### 状态与渲染链路

- 是否存在无意义重复渲染、重复派生、重复订阅
- 派生值是否被重复存入 state, 是否存在多份同语义数据源
- 组件、页面、store、hook 是否职责失衡, 是否把过多状态堆在单点
- 列表渲染是否缺少稳定 key, 是否在渲染期做重计算和数据转换

### 异步与交互

- 是否存在重复请求、无取消请求、竞态覆盖和失控重试
- 是否把 CPU 密集逻辑、布局计算或复杂数据处理塞进主线程
- 输入、滚动、resize 等高频事件是否缺少节流、去抖或缓存
- 是否把 IO、副作用、视图状态和业务状态混在一个长链路里

### 结构与边界

- 是否把所有逻辑堆进单一页面组件、单一 store 或单一 hook
- 模块边界、数据边界、缓存边界是否清晰
- 是否存在只为局部问题引入的过重抽象
- 命名、职责和目录结构是否直观表达业务和依赖关系

### 性能与资源风险

- 热路径是否频繁创建对象、数组、字符串或闭包
- 是否存在无意义 `map` / `filter` / `reduce` 链式复制
- 是否高频执行 `JSON.parse` / `JSON.stringify`、深拷贝或展开复制
- 是否缺少请求合并、去重、缓存、memo 边界和结果复用

## 工具建议

- React DevTools Profiler
- Chrome Performance Panel
- Lighthouse
- Node.js `--prof` / `clinic`
