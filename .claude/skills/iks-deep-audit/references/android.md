# Android Deep Audit

## 重点关注

- 崩溃、ANR、生命周期错用和状态恢复失败
- 主线程阻塞、渲染链路卡顿、启动链路过重
- 并发调度混乱、重复初始化和资源浪费
- 数据流耦合、层次失真和模块边界模糊

## 核心检查项

### 崩溃、ANR 与生命周期风险

- 是否存在主线程阻塞、同步等待、耗时初始化或生命周期错配
- Activity、Fragment、ViewModel、Service 的职责和恢复路径是否清晰
- 是否存在重复注册、重复观察、重复初始化和泄漏风险
- 配置变更、进程重建、页面切换后是否可能出现脏状态或异常恢复

### UI 与渲染链路

- 是否存在无意义重绘、过深布局层级或频繁 `requestLayout`
- RecyclerView / 列表是否重复绑定、重复计算或短生命周期对象过多
- 图片加载、动画、滚动、输入响应是否被主线程任务阻塞
- 是否把渲染状态、业务状态、数据转换混在单一 UI 链路中

### 并发与异步模型

- 协程作用域、取消路径、线程切换和回调链是否清晰
- 是否滥用 `Dispatchers.Main`, 是否把 IO 和重计算留在 UI 链路
- 是否存在后台任务拆得过碎、线程切换过多或结果回传混乱
- Flow / LiveData / Rx 链路是否存在重复订阅、重复发射或竞态覆盖

### 数据与结构边界

- 数据层是否重复拉取、重复映射、重复排序和重复缓存
- Room / SQLite / 网络请求是否在关键链路过重
- 展示层、状态层、数据转换层、数据访问层是否耦合过紧
- 模块边界、目录层次、命名语义是否足够直观

### 性能与资源风险

- 热路径是否频繁创建对象、字符串、集合和临时包装对象
- 是否缺少缓存、对象复用、分页、增量更新或 lazy 初始化
- 是否存在高成本日志、调试输出或无意义后台轮询
- 启动路径是否包含非必要初始化和阻塞式依赖

## 工具建议

- Android Studio Profiler
- Layout Inspector
- Macrobenchmark
- Perfetto / Systrace
