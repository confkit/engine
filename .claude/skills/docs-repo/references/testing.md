# Testing Template

## 适用场景

用于测试计划、场景矩阵、验收路径和回归范围说明, 不用于记录流程状态.

## 文件建议

- 目录下固定保留一个简洁的 `overview.md`, 只写范围、导航和阅读顺序
- 文件名使用英文 slug, 例如 `login-testing.md`, `payment-regression.md`
- 一个文件聚焦一个功能域或一组强相关场景

## 推荐结构

```md
# <testing-title>

## 测试范围

## 环境准备
| 项目 | 值 |
| --- | --- |
| 环境 | ... |
| 依赖 | ... |

## 场景矩阵
| 场景 | 前置条件 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| ... | ... | ... | ... |

## 关键用例
### <case-title>
1. ...
2. ...

预期结果:
- ...

## 不测范围
- ...
```

## 编写约束

- testing 先写环境准备和场景矩阵, 再补关键用例细节
- 关键场景必须写清前置条件、操作和预期结果
- 预期结果必须可判断, 不使用模糊表达
- 若步骤较长, 放到“关键用例”小节, 不压成单行字段
- testing 文档不承载状态流转, 不引入 `status`
