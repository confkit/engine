# Basic Documentation Sync

实施过程中, 必须保持文档与代码同步. 文档更新不是可选项, 是代码交付的一部分.

## 同步检查清单

修改代码前, 检查以下文档是否需要同步更新:

### 关联检查

- 本次修改涉及哪些模块/功能?
- 这些模块/功能是否有对应的设计文档?

### 设计文档

- 架构变更时更新对应设计文档
- 接口变更时更新 API 文档
- 流程变更时更新流程图和说明

### Todo 文档

- 任务完成时更新 todo 状态
- 新增任务时追加到 todo
- 发现新问题时补充到 todo

### 测试文档

- 新增功能时补充测试用例
- 变更逻辑时更新预期结果

### README

- 模块入口变更时更新
- 使用方式变更时更新
- 关键配置变更时更新

### CLAUDE.md

- 项目结构调整时更新目录说明
- 架构或技术栈变化时更新协作约定
- 构建、部署、核心依赖变化时同步更新

## 工作原则

- **修改前检查** - 开始编码前先确认需要更新哪些文档
- **同步更新** - 代码和文档在同一批次提交
- **完整性** - 不允许代码已合并但文档未更新的情况
